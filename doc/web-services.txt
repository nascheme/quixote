Implementing Web Services with Quixote
======================================

This document shows how to implement a Web service using Quixote.


An XML-RPC Service
------------------

XML-RPC is the simplest protocol commonly used to expose a Web service.  It
has a few basic data types (integers, floats, strings, dates) and a couple of
aggregate types (arrays and structs).  Python's standard-library
``xmlrpc.client`` module converts between Python values and XML-RPC types:

==============  ========================
XML-RPC Type    Python Type or Class
--------------  ------------------------
<int>           int
<double>        float
<string>        str
<array>         list
<struct>        dict
<boolean>       bool
<base64>        xmlrpc.client.Binary
<dateTime>      xmlrpc.client.DateTime
==============  ========================


Making XML-RPC Calls
--------------------

To call a service, create an ``xmlrpc.client.ServerProxy`` pointing at the
service URL and call methods on it::

    >>> import xmlrpc.client
    >>> s = xmlrpc.client.ServerProxy('http://www.example.com/rpc')
    >>> s.get_time()
    <DateTime '20260714T09:48:43' at ...>

The proxy marshals the call into an XML-RPC request, sends it, and unmarshals
the response.


Writing a Quixote Service
-------------------------

``quixote.util`` provides ``xmlrpc(request, func)``, which processes the body
of an XML-RPC request.  ``request`` is the ``HTTPRequest`` object Quixote
passes to every function it invokes.  ``func`` is a function you supply that
receives the name of the XML-RPC method being called and a tuple of the
method's parameters.  If ``func`` raises an exception, ``xmlrpc()`` catches it
and returns a ``Fault`` to the caller.

Here is a handler with a single method, ``get_time()``.  First expose a URL
for the service::

    import time
    import xmlrpc.client
    from quixote.directory import Directory
    from quixote.util import xmlrpc
    from quixote import get_request

    class RPCDirectory(Directory):

        _q_exports = ['rpc']

        def rpc(self):
            return xmlrpc(get_request(), rpc_process)

Placed in the ``__init__.py`` of your application package, this exposes
``http://<hostname>/rpc`` as the access point.

Next, fill in ``rpc_process()``, which receives the method name and
parameters and dispatches to the right code::

    def rpc_process(meth, params):
        if meth == 'get_time':
            # params is ignored
            now = time.gmtime(time.time())
            return xmlrpc.client.DateTime(now)
        else:
            raise RuntimeError("Unknown XML-RPC method: %r" % meth)

If the method name is not supported, the ``else`` clause raises
``RuntimeError``; Quixote's ``xmlrpc()`` reports it to the caller as an
XML-RPC fault with error code 1.

As you add methods, the ``if`` statement grows more branches.  You may be
tempted to pass the method name to ``getattr()`` to select a method
dynamically; that works but is easy to get wrong from a security standpoint,
since a remote caller could reach unintended attributes.  An explicit
dispatch table is safer, because only the names you list are callable::

    def echo(*params):
        return params

    def get_time():
        now = time.gmtime(time.time())
        return xmlrpc.client.DateTime(now)

    METHODS = {'echo': echo, 'get_time': get_time}

    def rpc_process(meth, params):
        func = METHODS.get(meth)
        if func is None:
            raise RuntimeError("Unknown XML-RPC method: %r" % meth)
        return func(*params)

This scales better than a long ``if... elif... else`` chain while keeping the
set of callable methods explicit.
